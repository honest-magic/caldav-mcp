2. Create `src/security/keychain.ts`:
   - Import `setPassword`, `getPassword`, `deletePassword` from `cross-keychain`
   - Import `config` from `../config.js`
   - `saveCredentials(accountId: string, secret: string): Promise<void>` — calls `setPassword(config.serviceName, accountId, secret)`
   - `loadCredentials(accountId: string): Promise<string | null>` — calls `getPassword`, catches errors, returns null on failure, logs with `console.error`

3. Create `src/security/oauth2.ts`:
   - Import `loadCredentials` and `saveCredentials` from `./keychain.js`
   - Import `AuthError` from `../errors.js`
   - Export `OAuth2Tokens` interface: `{ clientId: string, clientSecret: string, refreshToken: string, accessToken?: string, expiryDate?: number, tokenUrl: string }` (note: `tokenUrl` not `tokenEndpoint` — matches tsdav credential naming)
   - `getValidAccessToken(accountId: string): Promise<string>`:
     a. Load credentials from keychain via `loadCredentials(accountId)`
     b. If null, throw `new AuthError('No credentials found for account ${accountId}')`
     c. `JSON.parse(data)` into `OAuth2Tokens`; if parse fails, return data as-is (legacy plain password)
     d. If no `clientId` or `refreshToken`, return data as-is (plain password)
     e. If `accessToken` exists and `expiryDate` is > `Date.now() + 60000`, return `accessToken`
     f. Otherwise refresh: POST to `tokens.tokenUrl` with form-encoded body `{ client_id, client_secret, refresh_token, grant_type: 'refresh_token' }`
     g. If response not ok, throw `new AuthError('Failed to refresh token: ${status} ${errText}')`
     h. Update `tokens.accessToken`, `tokens.expiryDate` (from `expires_in * 1000 + Date.now()`), `tokens.refreshToken` if rotated
     i. Save updated tokens to keychain via `saveCredentials(accountId, JSON.stringify(tokens))`
     j. Return `tokens.accessToken!`

1. **Add `saveAccount` to `src/config.ts`** (append to existing file created by Plan 01):

```typescript
export async function saveAccount(account: CalDAVAccount): Promise<void> {
  const dir = path.dirname(ACCOUNTS_PATH);
  await fs.mkdir(dir, { recursive: true });
  let accounts: CalDAVAccount[] = [];
  try {
    const raw = await fs.readFile(ACCOUNTS_PATH, 'utf-8');
    accounts = JSON.parse(raw);
  } catch (err: any) {
    if (err.code !== 'ENOENT') throw err;
  }
  // Replace existing account with same id, or append
  const idx = accounts.findIndex(a => a.id === account.id);
  if (idx >= 0) {
    accounts[idx] = account;
  } else {
    accounts.push(account);
  }
  await fs.writeFile(ACCOUNTS_PATH, JSON.stringify(accounts, null, 2), 'utf-8');
  resetConfigCache(); // Invalidate cached accounts
}
```

Note: `fs` here is `node:fs/promises` (should already be imported in config.ts from Plan 01). Validate the account with `calDAVAccountSchema.parse(account)` before saving — throw `ValidationError` on failure.

**Private state:**
- `private clients: Map<string, CalDAVClient> = new Map()` — keyed by account.id

**Method: `async initialize(): Promise<void>`**
- Call `getAccounts()`
- For each account, create `new CalDAVClient(account)`, call `client.connect()`, store in `this.clients` map
- If connect fails for an account, log via `console.error` and continue with remaining accounts (don't fail the whole server)

**Method: `async listCalendars(accountId?: string): Promise<CalendarSummary[]>`**
- If `accountId` provided, get that client only; if not found, throw `new ValidationError('Account not found: ${accountId}')`
- If `accountId` not provided, iterate all clients
- For each client, call `client.fetchCalendars()`
- Map each `DAVCalendar` to `CalendarSummary`: `{ url: cal.url, displayName: cal.displayName ?? 'Untitled', ctag: cal.ctag ?? null, syncToken: cal.syncToken ?? null, accountId }`
- Return flat array of all CalendarSummary objects

**Method: `async listEvents(calendarUrl: string, startDate: string, endDate: string, accountId?: string): Promise<EventSummary[]>`**
- Resolve account: if `accountId` provided use that; otherwise search all clients for the calendar URL
- Find the calendar object by URL from `client.fetchCalendars()`
- Convert user-supplied dates to ISO 8601 UTC for tsdav's timeRange:
  ```typescript
  const start = DateTime.fromISO(startDate).toUTC().toISO();
  const end = DateTime.fromISO(endDate).toUTC().toISO();
  ```
  If either parse fails (invalid date), throw `new ValidationError('Invalid date format: ${startDate}')`
- Call `client.fetchCalendarObjects(calendar, { start, end })`
- For each `DAVObject` with `.data`:
  - Parse with `parseICS(obj.data)`
  - Map to `EventSummary`: `{ uid: parsed.uid, url: obj.url, etag: obj.etag ?? null, summary: parsed.summary, start: parsed.start, end: parsed.end, accountId, calendarUrl }`
  - Client-side date filter as safety net: skip events whose start time falls outside the requested range (defensive against servers that ignore REPORT time-range)
- Return array of EventSummary

**Method: `async readEvent(eventUrl: string, calendarUrl: string, accountId?: string): Promise<ParsedEvent>`**
- Resolve account same as listEvents
- Find calendar by URL
- Call `client.fetchSingleObject(calendar, eventUrl)`
- If null, throw `new ValidationError('Event not found: ${eventUrl}')`
- Parse with `parseICS(obj.data)` and return

**Method: `async registerOAuth2Account(params: { accountId: string; serverUrl: string; username: string; clientId: string; clientSecret: string; refreshToken: string; tokenUrl: string; name?: string }): Promise<{ accountId: string; message: string }>`**
- Construct a `CalDAVAccount` object: `{ id: params.accountId, name: params.name ?? params.accountId, serverUrl: params.serverUrl, authType: 'oauth2' as const, username: params.username }`
- Save credentials to keychain: `await saveCredentials(params.accountId, JSON.stringify({ clientId: params.clientId, clientSecret: params.clientSecret, refreshToken: params.refreshToken, tokenUrl: params.tokenUrl }))`
- Save account to config: `await saveAccount(account)`
- Attempt to connect: create `new CalDAVClient(account)`, call `client.connect()`, store in `this.clients` map
- If connect succeeds, return `{ accountId: params.accountId, message: 'Account registered and connected successfully' }`
- If connect fails, still keep the saved account/credentials (they might work later), return `{ accountId: params.accountId, message: 'Account registered but connection test failed: ${error.message}. Credentials saved — retry by restarting server.' }`

**Method: `getConnectedAccountIds(): string[]`**
- Return `Array.from(this.clients.keys())`

**Class: CalDAVMCPServer**

Constructor:
- Create `this.server = new Server({ name: 'caldav-mcp-server', version: '0.1.0' }, { capabilities: { tools: {} } })`
- Create `this.calendarService = new CalendarService()`
- Call `this.setupToolHandlers()`
- Set `this.server.onerror = (error) => console.error('[MCP Error]', error)`

**Method: `private getTools()`** — returns array of 5 tool definitions:

1. **list_calendars**:
   - description: "List all calendars across configured CalDAV accounts. Returns calendar names, URLs, and sync tokens."
   - inputSchema: `{ type: 'object', properties: { account: { type: 'string', description: 'Optional account ID to filter by. Defaults to all accounts.' } }, required: [] }`

2. **list_events**:
   - description: "List calendar events within a date range. Returns event summaries with UIDs, times, and URLs."
   - inputSchema: `{ type: 'object', properties: { calendarUrl: { type: 'string', description: 'Calendar URL (from list_calendars)' }, startDate: { type: 'string', description: 'Start of range, ISO 8601 (e.g. 2024-03-01 or 2024-03-01T00:00:00)' }, endDate: { type: 'string', description: 'End of range, ISO 8601 (e.g. 2024-03-31 or 2024-03-31T23:59:59)' }, account: { type: 'string', description: 'Optional account ID' } }, required: ['calendarUrl', 'startDate', 'endDate'] }`

3. **read_event**:
   - description: "Read full details of a calendar event including attendees, location, description, and recurrence rule."
   - inputSchema: `{ type: 'object', properties: { eventUrl: { type: 'string', description: 'Event URL (from list_events)' }, calendarUrl: { type: 'string', description: 'Calendar URL containing the event' }, account: { type: 'string', description: 'Optional account ID' } }, required: ['eventUrl', 'calendarUrl'] }`

4. **parse_ics**:
   - description: "Parse raw iCalendar (.ics) text into structured event data. No server connection needed — works with .ics content from email attachments or files."
   - inputSchema: `{ type: 'object', properties: { icsData: { type: 'string', description: 'Raw iCalendar (.ics) text content' } }, required: ['icsData'] }`

5. **register_oauth2_account** (per locked decision D-03: "Account setup via JSON config file + register_oauth2_account tool — consistent with mail_mcp"):
   - description: "Register a new OAuth2 CalDAV account. Saves credentials to OS keychain and account config to accounts.json. Use this after obtaining OAuth2 tokens from the provider's consent flow."
   - inputSchema:
     ```
     { type: 'object', properties: {
       accountId: { type: 'string', description: 'Unique account identifier (e.g. "google-personal")' },
       serverUrl: { type: 'string', description: 'CalDAV server URL (e.g. "https://apidata.googleusercontent.com/caldav/v2")' },
       username: { type: 'string', description: 'Account username or email address' },
       clientId: { type: 'string', description: 'OAuth2 client ID from provider' },
       clientSecret: { type: 'string', description: 'OAuth2 client secret from provider' },
       refreshToken: { type: 'string', description: 'OAuth2 refresh token obtained from consent flow' },
       tokenUrl: { type: 'string', description: 'OAuth2 token endpoint URL (e.g. "https://oauth2.googleapis.com/token")' },
       name: { type: 'string', description: 'Optional human-friendly account name' }
     }, required: ['accountId', 'serverUrl', 'username', 'clientId', 'clientSecret', 'refreshToken', 'tokenUrl'] }
     ```

**Method: `private setupToolHandlers(): void`**
- Register `ListToolsRequestSchema` handler: returns `{ tools: this.getTools() }`
- Register `CallToolRequestSchema` handler with switch on `request.params.name`:
  - `'list_calendars'`: call `this.calendarService.listCalendars(args.account)`, return JSON result
  - `'list_events'`: call `this.calendarService.listEvents(args.calendarUrl, args.startDate, args.endDate, args.account)`, return JSON result
  - `'read_event'`: call `this.calendarService.readEvent(args.eventUrl, args.calendarUrl, args.account)`, return JSON result
  - `'parse_ics'`: call `parseICS(args.icsData)`, return JSON result (no server needed)
  - `'register_oauth2_account'`: call `this.calendarService.registerOAuth2Account(args)`, return JSON result
  - default: throw `new McpError(ErrorCode.MethodNotFound, 'Unknown tool: ${name}')`
- Wrap each case in try/catch:
  - If error is `CalDAVMCPError`, return `{ content: [{ type: 'text', text: JSON.stringify({ error: error.message, code: error.code }) }], isError: true }`
  - If error is unknown, return `{ content: [{ type: 'text', text: JSON.stringify({ error: 'Internal error', details: String(error) }) }], isError: true }`

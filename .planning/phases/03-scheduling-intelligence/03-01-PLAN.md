    - Non-recurring event outside window produces no BusyPeriod
    - All-day event (VALUE=DATE) produces UTC midnight-to-midnight BusyPeriod for that date
    - Recurring RRULE (e.g. FREQ=DAILY;COUNT=5) within window produces correct number of BusyPeriods
    - EXDATE on a recurring event excludes that occurrence (5 daily - 1 EXDATE = 4 periods)
    - RECURRENCE-ID override replaces the original occurrence time with overridden time
    - Unbounded RRULE (no COUNT/UNTIL) stops at window end, does not loop infinitely
    - Events in non-UTC timezone (e.g. America/Chicago) produce DST-correct epoch values
    - Multiple ICS objects with same UID are grouped (master + exceptions)
  </behavior>
  <action>
    Create `src/utils/recurrence-expander.ts` following the verified code from RESEARCH.md Pattern 1.

    **Exports:**
    - `BusyPeriod` interface: `{ startMs: number; endMs: number }`
    - `icalTimeToMs(time: ICAL.Time, tzid: string): number` -- uses luxon DateTime.fromISO with TZID for conversion. All-day (isDate) uses UTC midnight. Floating uses 'local'.
    - `expandToBusyPeriods(allICSObjects: string[], windowStartMs: number, windowEndMs: number): BusyPeriod[]`

    **expandToBusyPeriods implementation (per RESEARCH.md):**
    1. Parse all ICS objects, extract VEVENTs
    2. Group by UID -- separate masters (no recurrence-id) from exceptions (has recurrence-id)
    3. For each master:
       a. Create ICAL.Event, call relateException() for each matching exception
       b. Extract TZID from master DTSTART property parameter, fallback 'UTC'
       c. If non-recurring: compute single BusyPeriod, check window, push if overlapping
       d. If recurring: call event.iterator() with NO arguments (critical -- per Pitfall 1), loop with getOccurrenceDetails(), extract TZID from occurrence's DTSTART property, convert via icalTimeToMs, break when startMs >= windowEndMs, skip when endMs <= windowStartMs
    4. Handle null endDate: use startDate as end (zero-duration)

    **Critical anti-patterns (from RESEARCH.md):**
    - NEVER pass startDate to event.iterator() -- breaks RECURRENCE-ID matching
    - NEVER use ICAL.RecurExpansion directly -- does not apply RECURRENCE-ID overrides
    - NEVER use time.toUnixTime() without VTIMEZONE -- use luxon with TZID string instead

    Write tests first (RED), then implement (GREEN). Test file uses vitest. Construct ICS strings inline in tests for full control.
